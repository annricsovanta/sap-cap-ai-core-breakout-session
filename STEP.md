# Step 3 — Business Partner External Service

## What we build
Connect the CAP application to the live SAP S/4HANA Business Partner API sandbox. Business Partner data is fetched in real-time — nothing is stored locally.

## Key concepts
- Importing an external OData service via `cds import`
- Projecting an external entity into a CAP service
- Delegating READ requests to the remote API
- Injecting API credentials via environment variables

## Instructions

### 1. Set up credentials
Copy `.env.example` to `.env` and fill in your SAP API Business Hub API key:
```
BP_API_KEY=your-key-here
```
Get your key at [SAP API Business Hub](https://api.sap.com).

### 2. Install and start
```sh
npm install
cds watch
```

### 3. Test with the REST client
Open `rest-client/demo-requests.http` and run the Business Partners request. You should see live BP data from the S/4 sandbox.

### 4. Explore the external service
- `srv/external/API_BUSINESS_PARTNER.cds` was generated by `cds import` from the API's EDMX metadata
- `srv/business-partner-service.cds` projects the external `A_BusinessPartner` entity
- `srv/business-partner-service.js` delegates every READ to the remote API and injects the API key header

## What's new in this step
