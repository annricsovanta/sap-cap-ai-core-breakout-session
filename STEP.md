# Step 2 — Insurance Service

## What we build
A CAP OData service that exposes the insurance domain model, plus a custom `after READ` handler that shows how to add business logic in CAP.

## Key concepts
- CDS service definition with projections
- CAP service handlers (`after READ`)
- OData v4 API auto-generated by CAP
- `$expand` to navigate associations

## Instructions

### 1. Install and start
```sh
npm install
cds watch
```

### 2. Test with the REST client
Open `rest-client/demo-requests.http` and run the request — you should see policy applications with their expanded policy data.

### 3. Observe the handler
Open `srv/insurance-service.js`. The `after READ` handler sets a default `status` of `pending` for any application that has none. Remove it temporarily and re-run to see the difference.
